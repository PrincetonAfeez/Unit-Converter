This is a strict scientific unit converter in Python with a layered architecture. The core domain models units, quantities, and dimensions separately from parsing and CLI concerns. The converter performs dimensional compatibility checks mathematically rather than by string matching, and unit definitions are loaded from data files instead of being hardcoded
a strict unit converter that supports:

base dimensions for a small set of categories:
length
mass
time
speed
force
strict compatibility checking
allow m -> km
reject m -> kg
compound units:
m/s
kg*m/s^2
metric prefixes:
milli, centi, kilo
small CLI:
convert 5 km to m
convert 10 m/s to km/h
unit definitions loaded from JSON
tests for parsing, compatibility, and conversion

Best architecture for this version

Use 3 layers, but keep each one tiny.

core/

Pure logic only.

Files:

units.py
quantity.py
registry.py
converter.py
exceptions.py

Small parser for unit strings.

Files:
